User Acceptance Testing with Jenkins, Selenium, Docker, and Kubernetes

Project Overview

In this project, we are tasked with conducting User Acceptance Testing (UAT) for an application that is managed and deployed through a series of automated processes. The key tools and technologies used in this workflow include Jira, Jenkins, Selenium IDE, Docker, Kubernetes, and AWS.

1. Jira: Project Management and Test Planning

Jira is used as the central platform for managing the project, where all the test plans, user stories, and test cases are documented. Specifically, we have 42 test cases listed in Jira, which need to be executed as part of the User Acceptance Testing by the deadline.

2. Selenium IDE: Test Execution

Selenium IDE is a browser-based tool used to automate the execution of these test cases. We use Selenium IDE to load and execute scripts directly in the browser. Some tests pass, while others fail, indicating the need for more advanced automation.

3. Jenkins: Continuous Integration and Automation

Jenkins is the automation server that orchestrates the CI/CD pipeline for building, testing, and deploying the application. It triggers automated test runs, integrates with Selenium tests, and builds and packages the application into a Docker container.

4. Docker: Containerization

Docker is used to package the application and its dependencies into containers, making it portable and easy to run across any environment. Jenkins packages the application and pushes the Docker images to a registry, ready for deployment.

5. Kubernetes: Orchestration and Scaling

Kubernetes handles the orchestration and management of Docker containers, ensuring that they are deployed across multiple nodes, scaled based on demand, and maintained for high availability.

6. AWS: Cloud Infrastructure

The entire application is hosted on AWS, providing the scalability, performance, and reliability needed to run the application. Kubernetes runs on AWS, leveraging its infrastructure to scale and manage the application efficiently.

Blue Ocean: A Better Jenkins Experience

Blue Ocean is a modern and user-friendly interface for Jenkins pipelines. It offers visual pipeline management, step-by-step troubleshooting, and an intuitive UI that makes it easier to manage complex CI/CD pipelines.

Pipeline Stages Explained

In Jenkins, pipelines consist of multiple stages:

  • Checkout: Pulls the latest code from a version control system like GitHub.
  • Build: Compiles or packages the code, often into a Docker image.
  • Test: Runs automated tests to ensure the quality of the code.
  • Deploy: Deploys the code to a production environment, such as a Kubernetes cluster on AWS.

Blue Ocean visualizes these stages as green, yellow, or red boxes, indicating success, in-progress, or failed stages.
